Murder Mystery Party Game API

API Documentation

This documentation provides an overview of the available routes and data models for the Murder Mystery Party Game API.

Throughout the project, this documentation may be used as a reference and a guide. It may be referred to whenever information or more details are required on how to implement the routes or models to the database.


Routes

In this section, you will find detailed information about the different routes available in the API.
The API offers a variety of routes to work with characters and plot-progression documents. Each route is associated with a specific HTTP verb and URL, allowing you to perform CRUD (Create, Read, Update, and Delete) actions on the data.


Character routes

HTTP verb URL Request body Action
GET /api/characters (empty) Returns all the characters in JSON format
GET /api/character/:characterId (empty) Returns the specified character by id
POST /api/characters JSON Creates a new character
PUT /api/character/:characterId JSON Updates the specified character by id
DELETE /api/character/:characterId (empty) Deletes the specified character by id

Investigation Notes

HTTP verb URL Request body Action
GET /api/investigation-notes (empty) Returns all the investigation notes in JSON format
GET /api/investigation-notes/:investigation-notesId (empty) Returns the investigation notes for a single character in JSON format
POST /api/investigation-notes JSON Creates a new investigation note with a respective character id
PUT /api/investigation-notes/:investigation-notesId JSON Updates the specified student by id
DELETE /api/investigation-notes/:investigation-notesId (empty) Deletes the specified cohort by id

Models

The Models section holds information about the data models for your database. It outlines the structure of the documents in the database, providing you with a clear understanding of how your data should be organized.


Character Model

Field Data Type Description
characterID String Unique identifier for the character. Required.
Name String Name of the character. Required.
backstory String Character backstory - who are they, how are they related to the victim.

Investigation Notes Model

Field Data Type Description
Motive String Reason why they could be guilty. Required.
Clue String A prop or action that confirms the motive. Required.
Quote String Something memorable a character said. Required, unique.
